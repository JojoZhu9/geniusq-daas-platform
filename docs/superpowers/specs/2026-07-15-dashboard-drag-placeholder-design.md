# 仪表盘拖拽空白落点提示设计

## 目标

优化“我的仪表盘”的卡片移动交互：移除卡片底部的“移动卡片”按钮，只保留拖动手柄；拖动到空白栅格区域时，显示与被拖卡片相同尺寸的蓝色虚线占位框，让用户在松手前明确看到最终落点。

## 范围

- 删除“移动卡片”按钮及其左右切换处理函数。
- 保留“⠿ 拖动”、放大/恢复尺寸和移除功能。
- 拖到已有卡片时，继续使用现有蓝色目标卡片高亮并在松手后交换位置。
- 拖到空白区域时，显示蓝色虚线占位框并在松手后移动卡片。
- 拖拽结束、取消或移出仪表盘后，清除所有拖拽提示。
- 不修改后端接口、数据库结构和公开分享页。

## 交互设计

仪表盘继续使用现有 12 列 CSS Grid。开始拖动后，前端根据指针相对仪表盘的位置计算目标列和目标行，并将结果保存为临时的空白落点状态 `{ x, y, w, h }`。

- 指针位于已有卡片上：清除空白落点状态，为目标卡片增加蓝色实线轮廓。
- 指针位于空白区域：清除目标卡片状态，在对应栅格位置渲染蓝色虚线占位元素。
- 占位元素的 `w/h` 直接取被拖卡片尺寸，因此放大卡片也能得到相同尺寸的提示框。
- 指针落在已有卡片实际占用的栅格上时，不显示空白框，避免同时出现两种提示。
- 松手时使用与预览相同的目标坐标保存布局，保证“看到的落点”与“保存的落点”一致。

## 状态与数据流

新增一个仅存在于页面内存的空白落点状态。指针移动时统一调用目标位置计算函数，预览和最终保存共享该计算结果。`finishDrag` 统一清除被拖卡片、目标卡片和空白落点三类状态。

布局保存仍调用现有 `PATCH /api/dashboards/{id}/layout`。已有卡片位置被占用时继续交换卡片；空白位置则只更新被拖卡片。

## 样式

空白占位框使用独立的栅格元素：透明浅蓝背景、2px 蓝色虚线边框、轻微蓝色阴影、圆角与卡片一致，并禁用指针事件，避免阻断拖拽事件。

## 验收与测试

- 页面中不再出现“移动卡片”按钮。
- 按住拖动手柄移向空白区域时，出现与源卡片相同栅格尺寸的占位框。
- 移向已有卡片时，占位框消失且目标卡片高亮。
- 在空白区域松手后发送正确的 `x/y/w/h` 并显示“布局已保存”。
- 拖拽取消或在仪表盘外松手后不残留占位框。
- 现有卡片交换、刷新持久化、放大避让和公开分享功能保持通过。
